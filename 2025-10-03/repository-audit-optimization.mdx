## 3. GitHub Metadata Optimization

### 3.1 Essential Metadata Checklist

For each repository, optimize:

#### Repository Settings
- [ ] **Repository Name:** Clear, descriptive, keyword-rich
- [ ] **Description:** Compelling one-liner (max 120 chars) with keywords
- [ ] **Website URL:** Link to docs, demo, or main site
- [ ] **Topics (5-8):** Relevant, searchable keywords
- [ ] **License:** MIT or Apache 2.0 (open source signal)
- [ ] **Social Preview Image:** Custom GitHub card (1280x640px)

#### README.md Structure
- [ ] **H1 Title:** Keyword-rich, descriptive
- [ ] **Badges:** Build status, coverage, version, downloads
- [ ] **Quick Start:** 3-5 commands to get running
- [ ] **Features:** Bullet list of capabilities
- [ ] **Installation:** Multiple package managers
- [ ] **Usage Examples:** Code snippets with syntax highlighting
- [ ] **API Reference:** Link or embed
- [ ] **Contributing:** Guidelines for contributors
- [ ] **License:** Clear license statement

### 5.2 GitHub Pages Technical Requirements

**Framework Options:**

1. **VitePress** (Recommended)
   - Fast, Vue-based
   - Built-in search
   - Markdown-centric
   - Excellent DX

2. **Docusaurus**
   - React-based
   - Feature-rich
   - Large community
   - MDX support

3. **MkDocs Material**
   - Python-based
   - Beautiful theme
   - Fast search
   - Easy setup

4. **Fumadocs** (Already used in docs/)
   - Next.js-based
   - Flexsearch
   - MDX-first
   - TypeScript

**Deployment Configuration:**

\`\`\`yaml
# .github/workflows/deploy-pages.yml
name: Deploy to GitHub Pages

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: pnpm/action-setup@v2
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'pnpm'

      - run: pnpm install
      - run: pnpm build:docs

      - uses: actions/upload-pages-artifact@v3
        with:
          path: ./dist

  deploy:
    needs: build
    runs-on: ubuntu-latest
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    steps:
      - uses: actions/deploy-pages@v4
        id: deployment
\`\`\`
