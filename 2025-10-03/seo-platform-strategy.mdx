Based on comprehensive research across 7 critical SEO/AEO domains, this document provides actionable recommendations for building maximum SEO/AEO capabilities into the .do platform architecture. The research reveals that success in 2025 requires optimization for both traditional search engines AND AI answer engines, with a focus on:

- **Technical Excellence**: llms.txt, JSON-LD, Core Web Vitals, mobile-first
- **AI Optimization**: AEO/GEO strategies for ChatGPT, Perplexity, Claude, Gemini

// Edge caching for static assets
export default {
  async fetch(request, env) {
    const cache = caches.default
    let response = await cache.match(request)

    if (!response) {
      response = await fetch(request)
      response = new Response(response.body, response)
      response.headers.set('Cache-Control', 'public, max-age=3600')
      await cache.put(request, response.clone())
    }

    return response
  }
}

### 4.2 Anchor Text Distribution (Safe Pattern)

For natural link profile:

- **Exact Match**: 10-15% (e.g., "cloudflare workers platform")
- **Partial Match**: 15-20% (e.g., "microservices platform")
- **Branded**: 30-40% (e.g., "services.do", "dot-do")
- **Generic**: 20-25% (e.g., "click here", "learn more", "this guide")
- **Naked URL**: 10-15% (e.g., "https://services.do")

**Over-Optimization Red Flags**:
- > 15% exact-match anchors
- Identical anchor text from multiple sources
- All links from same anchor pattern

**Key Principles**:

5. **No Sitewide Links**: Avoid footer/sidebar links across network

**Proposed Implementation**:

```typescript
// workers/seo/src/index.ts

export class SeoService extends WorkerEntrypoint {
  async researchKeywords(seed: string, options: KeywordOptions) {
    // Aggregate data from multiple APIs
    const [semrush, ahrefs] = await Promise.all([
      this.fetchSemrushData(seed),
      this.fetchAhrefsData(seed)
    ])

    // AI-powered clustering and intent analysis
    const clustered = await this.env.AI.run('@cf/meta/llama-2-7b-chat-int8', {
      messages: [{
        role: 'user',
        content: `Cluster these keywords by topic and intent: ${keywords}`
      }]
    })

    // Store in database
    await this.env.DB.insertKeywords(keywords, clusters)

    return { keywords, clusters, opportunities }
  }

  async trackRankings(keywords: string[]) {
    // Position tracking via SERPWatcher or custom
    const rankings = await this.checkRankings(keywords)

    // Alert on significant changes
    await this.env.QUEUE.send({
      type: 'ranking_change',
      keywords: rankings.filter(r => r.change > 3)
    })

    return rankings
  }

  async analyzeCompetitors(domain: string) {
    // Keyword gap analysis
    const gaps = await this.findKeywordGaps(domain)

    return {
      gap_keywords: gaps,
      opportunities: gaps.filter(k => k.difficulty < 40 && k.volume > 500)
    }
  }
}
```

## 9. Risk Mitigation

### SEO Risks & Mitigation Strategies

**Risk 1: Google Algorithm Updates**
- **Mitigation**: Focus on E-E-A-T, quality content, user value
- **Strategy**: Diversify traffic sources (AI, social, email)
- **Monitoring**: Weekly traffic reviews, quick response to drops

**Risk 2: AI Crawler Blocking**
- **Mitigation**: Allow major AI crawlers in robots.txt
- **Balance**: Block only admin/user data, not public content
- **Review**: Quarterly assessment of AI crawler strategy

**Risk 3: PBN Detection (Domain Network)**
- **Mitigation**: Transparent ownership, unique content, natural links
- **Avoidance**: No footer links, no identical templates, no link schemes
- **Documentation**: Public disclosure of domain relationships

**Risk 4: Content Quality at Scale**
- **Mitigation**: Editorial review process, quality thresholds
- **Automation**: AI-assisted, not AI-generated without review
- **Standards**: Minimum word count, E-E-A-T signals, original insights

**Risk 5: Negative SEO Attacks**
- **Mitigation**: Monthly backlink audits, disavow toxic links
- **Monitoring**: Ahrefs/SEMrush alerts for new backlinks
- **Response**: Disavow Tool for obvious spam
