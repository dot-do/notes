# Email Pattern Data Sources Research
**Date:** 2025-10-03
**Purpose:** Comprehensive research on email pattern data sources, APIs, and alternative approaches

## Executive Summary

This research covers commercial APIs (RocketReach, ZoomInfo, Hunter.io, Apollo.io, Clearbit, FullContact, Snov.io), open-source approaches, scraping strategies, and email verification services. Key findings show that most commercial APIs use credit-based pricing models ($39-50K+/year), provide confidence scores, and offer both enrichment and verification capabilities.

## 9. Email Pattern Formats

### Most Common Patterns (in order of frequency)

1. **first.last@domain.com** - Most standard format globally
   - Example: john.smith@company.com
   - Pros: Clear, unambiguous, professional
   - Cons: Longest format

2. **flast@domain.com** - First initial + last name
   - Example: jsmith@company.com
   - Pros: Shorter, still clear
   - Cons: Duplicates for common names

3. **first@domain.com** - First name only
   - Example: john@company.com
   - Pros: Personal, memorable
   - Cons: Only works for unique first names

4. **firstlast@domain.com** - Combined without separator
   - Example: johnsmith@company.com
   - Pros: Clean, simple
   - Cons: Harder to read

5. **first_last@domain.com** - Underscore separator
   - Example: john_smith@company.com
   - Pros: Alternative to period
   - Cons: Less common

6. **first-last@domain.com** - Hyphen separator
   - Example: john-smith@company.com
   - Pros: Another separator option
   - Cons: Rare in business

### Pattern Inference Strategy

**Method:**
1. Find 2-3 known emails at target company
2. Identify common pattern
3. Generate variations for target person
4. Test with verification API
5. Use highest confidence result

**Common Pattern Variations:**
```javascript
const patterns = [
  '{first}.{last}@{domain}',
  '{f}{last}@{domain}',
  '{first}@{domain}',
  '{first}{last}@{domain}',
  '{first}_{last}@{domain}',
  '{first}-{last}@{domain}',
  '{last}.{first}@{domain}',
  '{last}{f}@{domain}',
]
```

## 10. Email Verification APIs

### Why Verification Matters
- Reduce bounce rates (>5% damages sender reputation)
- Avoid spam traps and disposable emails
- Validate syntax, DNS, and SMTP
- Detect catch-all domains

### Verification Process

**3-Step Validation:**
1. **Syntax Validation:** RFC 5322 compliance, typo detection
2. **MX Record Verification:** DNS records exist for domain
3. **SMTP Server Verification:** Mailbox exists and accepts mail

### Top Verification APIs

**Emailable** - Top choice per Resend
- Focus on accuracy
- Modern API

**Bouncer** - Best uptime/accuracy
- 100% uptime guarantee
- <2% unknown results
- 99% accuracy guarantee
- Bias against false negatives (conservative)

**MillionVerifier** - Best for general validation
- Cost-effective
- Accurate
- Simple integration

**Instantly** - Best for catch-all verification
- Specialized in catch-all domains
- Outreach-focused

**SendGrid** - Enterprise option
- Doesn't use broken SMTP-handshake
- Avoids spammer-associated tactics
- Trusted infrastructure

### Other Notable Services
- ZeroBounce
- Hunter (also does verification)
- AbstractAPI
- DeBounce
- Mailgun

### Key Comparison Factors
- **Accuracy rates** and unknown percentages
- **SMTP validation** methods
- **Catch-all handling** (difficult to verify)
- **Disposable email** detection
- **Bounce rate** guarantees
- **Pricing models** (per-validation vs. subscription)
- **Integration options** (real-time vs. bulk)

- **Public Data:** Generally legal to scrape publicly available data

- **Compliance:** Must follow GDPR, CAN-SPAM, CCPA

**Against ToS:**

**Best Practice:**

- Follow GDPR consent requirements

- Have legitimate business reason

function inferEmailPattern(firstName, lastName, domain, knownEmails) {
  // 1. Find known employees at company
  const companyEmails = knownEmails.filter(e => e.domain === domain)

  // 2. Detect pattern
  const pattern = detectPattern(companyEmails)

## 14. Technical Implementation

### API Integration Pattern

```typescript
// Example: Hunter.io Email Finder
interface EmailFinderResult {
  email: string
  score: number // 0-100
  verification: {
    status: 'valid' | 'accept_all' | 'unknown'
    date: string
  }
  sources: Array<{
    domain: string
    uri: string
    extracted_on: string
  }>
}

async function findEmail(
  firstName: string,
  lastName: string,
  domain: string
): Promise<EmailFinderResult> {
  const response = await fetch(
    `https://api.hunter.io/v2/email-finder?` +
    `domain=${domain}&` +
    `first_name=${firstName}&` +
    `last_name=${lastName}&` +
    `api_key=${process.env.HUNTER_API_KEY}`
  )

  const data = await response.json()
  return data.data
}
```

### Multi-Provider Fallback

```typescript
// Try multiple providers in order
const providers = [
  { name: 'hunter', cost: 0.1, call: hunterFindEmail },
  { name: 'snov', cost: 0.03, call: snovFindEmail },
  { name: 'apollo', cost: 0.15, call: apolloEnrich },
]

async function findEmailWithFallback(person, company) {
  for (const provider of providers) {
    try {
      const result = await provider.call(person, company)
      if (result.confidence > 80) {
        return { ...result, provider: provider.name, cost: provider.cost }
      }
    } catch (error) {
      console.warn(`${provider.name} failed:`, error)
      continue
    }
  }

  throw new Error('All providers failed')
}
```

### Pattern-Based Generation

```typescript
// Generate email variations
function generateEmailVariations(
  firstName: string,
  lastName: string,
  domain: string
): string[] {
  const f = firstName.toLowerCase()
  const l = lastName.toLowerCase()
  const fi = f[0]
  const li = l[0]

  return [
    `${f}.${l}@${domain}`,
    `${fi}${l}@${domain}`,
    `${f}@${domain}`,
    `${f}${l}@${domain}`,
    `${f}_${l}@${domain}`,
    `${f}-${l}@${domain}`,
    `${l}.${f}@${domain}`,
    `${l}${fi}@${domain}`,
  ]
}

// Verify all variations
async function findEmailByPattern(person, company) {
  const variations = generateEmailVariations(
    person.firstName,
    person.lastName,
    company.domain
  )

  const results = await Promise.all(
    variations.map(email => verifyEmail(email))
  )

  // Return highest confidence
  return results.reduce((best, current) =>
    current.confidence > best.confidence ? current : best
  )
}
```
