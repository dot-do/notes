## Executive Summary

Based on comprehensive research across 10+ SEO/AEO domains, this document provides an actionable implementation plan for building maximum SEO/AEO capabilities into the .do platform architecture. The research reveals that success in 2025 requires optimization for both traditional search engines AND AI answer engines.

### Key Insights

- **60-63% of searches** are now zero-click (answered on SERP)
- **13.14% of queries** trigger AI Overviews (up from 6.49% in Jan 2025)
- **527% increase** in AI-referred traffic (Jan-May 2025)
- **40% more citations** for sites with llms.txt
- **3.2x more AI citations** for content updated within 30 days

### Success Requirements

1. **Technical Excellence** - llms.txt, JSON-LD, Core Web Vitals, mobile-first
2. **AI Optimization** - AEO/GEO strategies for ChatGPT, Perplexity, Claude, Gemini
3. **Quality Content** - E-E-A-T signals, original research, comprehensive guides
4. **Natural Growth** - Editorial link building, Digital PR, organic backlinks
5. **Multi-Platform Tracking** - Monitor citations across all major AI platforms

## SEO/AEO Primitives & Data Sources

### 1. Technical SEO Infrastructure

**llms.txt Implementation**
- File at root of every domain
- Structured markdown with navigation
- AI-friendly content summary
- Benefits: 40% more AI citations

**robots.txt Management**
- Per-bot access controls
- Strategic AI crawler access
- Training vs search index rules

**Sitemap Generation**
- XML sitemap by content type
- Automatic updates on publish
- Submit to Search Console

**Core Web Vitals**
- LCP < 2.5s, INP < 200ms, CLS < 0.1
- Cloudflare Web Analytics (free)
- Real-time monitoring

## Parallelized Workstreams

### Workstream 1: SEO Crawler Detection Worker
**Timeline:** 3-4 days
**Priority:** P0 - Foundation

**Implementation:**
- Bot detection service (user agent parsing)
- robots.txt generator with per-bot rules
- Access control middleware
- Crawler log aggregation
- Real-time bot tracking

**Endpoints:**
- `POST /track/bot` - Log bot access
- `GET /bots/summary` - Bot activity summary
- `GET /robots.txt/generate` - Generate robots.txt
- `GET /bots/allowed` - Check bot access rules
- `PUT /bots/rules` - Update access controls

**Dependencies:**
- Analytics service (event tracking)
- DB service (bot logs)
