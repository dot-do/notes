## Part 6: GitHub Discussions Setup (30 minutes)

**Important:** Discussions are enabled per-repository, not organization-wide.

### Step 6.1: Enable Discussions on .do Repository

**Location:** https://github.com/dot-do/.do/settings

1. **Navigate to repository settings:**
   - Go to: https://github.com/dot-do/.do
   - Click "Settings" tab (requires admin access)

2. **Enable Discussions feature:**
   - Scroll to "Features" section
   - Find "Discussions" checkbox
   - Check "Discussions"
   - Page will reload

3. **Access Discussions settings:**
   - New "Discussions" tab appears in repository
   - Click "Discussions" tab
   - Click "Set up Discussions" if prompted

### Step 6.2: Configure Discussion Categories

**Default categories** are created automatically. **Customize them:**

1. **Go to:** https://github.com/dot-do/.do/discussions
2. **Click:** Categories (right sidebar)
3. **Edit/Add categories** as follows:

#### Category Configuration Matrix

| Emoji | Name | Description | Format |
|-------|------|-------------|--------|
| 📢 | Announcements | Official updates from the .do team | Announcement |
| 💬 | General | General discussion about .do | Open-ended |
| 💡 | Ideas | Share ideas for new features | Open-ended |
| ❓ | Q&A | Ask questions, get answers from the community | Q&A |
| 🎨 | Show and Tell | Share your projects built with .do | Open-ended |
| 🆘 | Help | Get help with issues and problems | Open-ended |
| 🏗️ | Architecture | Discuss technical architecture and design | Open-ended |
| 🔌 | Integrations | Talk about integrations and connectors | Open-ended |
| 💭 | Feedback | Provide feedback on .do platform | Open-ended |

**How to Add/Edit Categories:**
1. Click "Edit" next to category name
2. Update name, emoji, description, format
3. Click "Save"
4. Repeat for all categories

**Category Format Explanation:**
- **Announcement:** Only maintainers can create, comments allowed
- **Q&A:** Has "Mark as answer" feature, best for support questions
- **Open-ended:** Standard discussion format

### Step 6.3: Create Welcome Discussion

1. **Go to:** https://github.com/dot-do/.do/discussions
2. **Click:** "New discussion" button
3. **Select category:** Announcements
4. **Title:** Welcome to the .do Community! 🎉
5. **Body:** (Use template from implementation guide)
6. **Click:** "Start discussion"
7. **Pin discussion:**
   - Click "..." menu on discussion
   - Select "Pin discussion"
   - Discussion appears at top

## Common Issues and Solutions

### Issue: "Settings" tab not visible

**Solution:**
- Verify you have Owner or Admin role in organization
- Go to: https://github.com/orgs/dot-do/people
- Check your role
- If not Owner/Admin, ask organization owner for access

### Issue: Cannot update organization name

**Context:** Organization username (dot-do) is permanent
**Solution:**
- Use "Display name" field for branding (.do)
- Username remains dot-do for URLs

### Issue: Verified domain verification fails

**Solution:**
1. Check DNS propagation (use https://dnschecker.org)
2. Wait 5-10 minutes after adding TXT record
3. Verify TXT record format exactly matches GitHub's requirement
4. Contact GitHub Support if persistent issues

### Issue: Pinned repositories not updating

**Solution:**
1. Clear browser cache
2. Wait 5-10 minutes for GitHub cache to update
3. Try in incognito/private browsing mode
4. Verify repositories are public
