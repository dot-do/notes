```markdown

   ```bash
   git checkout -b add-api-do-to-category
   ```

3. Edit README.md
   - Add entry in alphabetical order
   - Match existing format exactly
   - Verify markdown rendering

4. Commit changes
   ```bash
   git add README.md
   git commit -m "Add api.do to [Category]"
   ```

5. Push to your fork
   ```bash
   git push origin add-api-do-to-category
   ```

6. Create Pull Request
   - Go to original repo on GitHub
   - Click "New Pull Request"
   - Select your branch
   - Use pre-written PR description
   - Submit!

   - Respond to feedback within 24 hours
