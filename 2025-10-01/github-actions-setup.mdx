## Overview

This guide covers the deployment of the Claude Code GitHub Actions workflow to all repositories in the dot-do organization. The workflow automatically processes issues labeled with `claude-code`, runs Claude Code to implement requested changes, and creates pull requests.

### Repositories Covered (12 total)

**Root Repository:**
- `.do` - Multi-repo parent (requires submodules)

**Core Service Repositories:**
- `api.services` - Monolithic API (migration source)
- `api` - Extracted API services
- `db` - Database layer
- `ai` - AI/ML features

**Application Repositories:**
- `agent` - AI code generation
- `app` - CMS & admin
- `site` - Public websites monorepo

**Supporting Repositories:**
- `sdk` - TypeScript SDK packages
- `do.industries` - Industries directory
- `docs` - Documentation site
- (Add any additional repositories here)

### Workflow Capabilities

- ‚úÖ Automatic issue processing via `claude-code` label
- ‚úÖ Priority detection from issue titles (`[P0]`, `[P1]`, `[P2]`)
- ‚úÖ Smart repository detection (submodules for root, none for sub-repos)
- ‚úÖ Automatic PR creation with metadata
- ‚úÖ Error handling and status updates
- ‚úÖ Concurrency control (one issue at a time)

## Organization-Level Setup

### Step 1: Configure Organization Secrets

Organization secrets are shared across all repositories, making management easier.

```bash
# Set organization-level secret for Anthropic API key
gh secret set ANTHROPIC_API_KEY \
  --org dot-do \
  --body "YOUR_ANTHROPIC_API_KEY_HERE"

# Verify the secret was created
gh secret list --org dot-do
```

**Alternative: Using GitHub Web UI**

1. Go to `https://github.com/organizations/dot-do/settings/secrets/actions`
2. Click "New organization secret"
3. Name: `ANTHROPIC_API_KEY`
4. Value: Your Anthropic API key
5. Repository access: "All repositories" or select specific ones
6. Click "Add secret"

### Step 2: Create Standard Labels

Create labels that will be used across all repositories:

```bash
# Function to create labels in a repository
create_labels() {
  local repo=$1

  # Claude Code trigger label
  gh label create "claude-code" \
    --repo "dot-do/$repo" \
    --color "7057ff" \
    --description "Trigger Claude Code automation" \
    --force

  # Priority labels
  gh label create "priority:P0" \
    --repo "dot-do/$repo" \
    --color "d73a4a" \
    --description "Critical priority - immediate attention" \
    --force

  gh label create "priority:P1" \
    --repo "dot-do/$repo" \
    --color "fbca04" \
    --description "High priority - should be addressed soon" \
    --force

  gh label create "priority:P2" \
    --repo "dot-do/$repo" \
    --color "0e8a16" \
    --description "Normal priority - standard timeline" \
    --force

  # Error handling label
  gh label create "claude-code-error" \
    --repo "dot-do/$repo" \
    --color "d93f0b" \
    --description "Claude Code encountered an error" \
    --force

  # Automation label
  gh label create "automated" \
    --repo "dot-do/$repo" \
    --color "bfdadc" \
    --description "Automatically generated by automation" \
    --force
}

# Create labels in all repositories
repos=(".do" "api.services" "api" "db" "ai" "agent" "app" "site" "sdk" "do.industries" "docs")

for repo in "${repos[@]}"; do
  echo "Creating labels in $repo..."
  create_labels "$repo"
done
```

## Troubleshooting

### Common Issues and Solutions

#### Issue: Workflow Not Triggering

**Symptoms:**
- Issue is labeled but workflow doesn't start
- No workflow runs appear in Actions tab

**Solutions:**

1. Verify workflow file exists:
   ```bash
   gh api repos/dot-do/REPO_NAME/contents/.github/workflows/claude-code.yml
   ```

2. Check workflow syntax:
   ```bash
   # Download and validate
   gh api repos/dot-do/REPO_NAME/contents/.github/workflows/claude-code.yml \
     --jq '.content' | base64 -d > /tmp/workflow.yml

   # Validate YAML syntax
   python3 -c "import yaml; yaml.safe_load(open('/tmp/workflow.yml'))"
   ```

3. Verify label exists:
   ```bash
   gh label list --repo dot-do/REPO_NAME | grep claude-code
   ```

4. Check repository permissions (Settings ‚Üí Actions ‚Üí General)

#### Issue: Authentication Errors

**Symptoms:**
- "Error: Could not authenticate with Claude API"
- Workflow fails at Claude Code step

**Solutions:**

1. Verify secret exists:
   ```bash
   gh secret list --repo dot-do/REPO_NAME | grep ANTHROPIC_API_KEY
   # or org level
   gh secret list --org dot-do | grep ANTHROPIC_API_KEY
   ```

2. Test API key manually:
   ```bash
   curl https://api.anthropic.com/v1/messages \
     -H "x-api-key: YOUR_API_KEY" \
     -H "anthropic-version: 2023-06-01" \
     -H "content-type: application/json" \
     -d '{"model":"claude-3-sonnet-20240229","max_tokens":1024,"messages":[{"role":"user","content":"Hello"}]}'
   ```

3. Recreate secret:
   ```bash
   gh secret set ANTHROPIC_API_KEY --repo dot-do/REPO_NAME
   ```

#### Issue: Permission Denied on PR Creation

**Symptoms:**
- "Error: Resource not accessible by integration"
- PR creation fails

**Solutions:**

1. Update workflow permissions:
   - Go to Settings ‚Üí Actions ‚Üí General
   - Enable "Read and write permissions"
   - Enable "Allow GitHub Actions to create and approve pull requests"

2. Verify token permissions in workflow logs

3. Check organization security settings

#### Issue: Submodule Errors in Root Repo

**Symptoms:**
- Submodule checkout fails
- Missing submodule files

**Solutions:**

1. Verify `.gitmodules` file exists:
   ```bash
   gh api repos/dot-do/.do/contents/.gitmodules
   ```

2. Test submodule checkout manually:
   ```bash
   git clone --recurse-submodules https://github.com/dot-do/.do.git /tmp/test-do
   ```

3. Update workflow to use specific submodule strategy:
   ```yaml
   - name: Checkout Code (with submodules for root repo)
     uses: actions/checkout@v4
     with:
       fetch-depth: 0
       submodules: recursive
       token: ${{ secrets.GITHUB_TOKEN }}
   ```

#### Issue: Claude Code Execution Fails

**Symptoms:**
- Claude Code command not found
- Execution errors

**Solutions:**

1. Verify Claude Code CLI installation step:
   ```yaml
   - name: Install Claude Code CLI
     run: npm install -g @anthropic-ai/claude-code
   ```

2. Check npm registry access:
   ```bash
   npm view @anthropic-ai/claude-code
   ```

3. Review Claude Code version compatibility

#### Issue: No Changes Detected

**Symptoms:**
- Workflow completes but no PR created
- "No changes made" comment on issue

**Solutions:**

1. Review issue description - ensure it's actionable
2. Check Claude Code prompt generation in workflow logs
3. Verify repository has files that can be modified
4. Review CLAUDE.md guidelines - ensure they're clear

## Advanced Configuration

### Custom Workflow Modifications

#### Per-Repository Customization

Each repository can customize its workflow while maintaining core functionality:

**Example: Extended Timeout for Large Repos**

```yaml
jobs:
  claude-code:
    timeout-minutes: 60  # Increase from 30 to 60 for api.services
```

**Example: Repository-Specific Labels**

```yaml
- name: Add Custom Labels
  run: |
    gh pr edit $PR_NUMBER \
      --add-label "backend" \
      --add-label "database"
```

**Example: Notification Integration**

```yaml
- name: Notify Slack
  if: steps.claude.outputs.changes_made == 'true'
  uses: slackapi/slack-github-action@v1
  with:
    webhook-url: ${{ secrets.SLACK_WEBHOOK }}
    payload: |
      {
        "text": "Claude Code created PR for issue #${{ github.event.issue.number }}"
      }
```

#### Multi-Stage Processing

For complex issues requiring multiple steps:

```yaml
- name: Multi-Stage Processing
  run: |
    # Stage 1: Analysis
    claude-code analyze --output /tmp/analysis.json

    # Stage 2: Implementation
    claude-code implement --plan /tmp/analysis.json

    # Stage 3: Testing
    claude-code test --verify
```

#### Integration with Existing CI/CD

Coordinate with existing workflows:

```yaml
- name: Trigger CI Pipeline
  run: |
    gh workflow run ci.yml \
      --ref ${{ steps.claude.outputs.branch }}
```

### Security Enhancements

#### Rate Limiting

Prevent API abuse:

```yaml
- name: Check Rate Limit
  run: |
    # Implement rate limiting logic
    ISSUE_COUNT=$(gh issue list --label claude-code --state open --json number --jq 'length')
    if [ $ISSUE_COUNT -gt 10 ]; then
      echo "Too many open claude-code issues. Please close some before creating new ones."
      exit 1
    fi
```

#### Issue Validation

Validate issue content before processing:

```yaml
- name: Validate Issue
  run: |
    BODY="${{ github.event.issue.body }}"
    if [ -z "$BODY" ] || [ ${#BODY} -lt 20 ]; then
      gh issue comment ${{ github.event.issue.number }} \
        --body "‚ùå Issue body is too short. Please provide more details."
      exit 1
    fi
```

#### Approval Workflow

Require approval before execution:

```yaml
- name: Wait for Approval
  uses: trstringer/manual-approval@v1

    secret: ${{ secrets.GITHUB_TOKEN }}

```
