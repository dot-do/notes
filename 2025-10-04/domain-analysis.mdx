## Recommendations

### 1. Unify Domain Configuration

Currently domains are split across two files with different purposes:
- **API Routing** (`api/src/routing/domains.ts`) - Maps domains to worker services
- **Site Config** (`site/apps/do.industries/domains.config.ts`) - Domain metadata for marketing site

**Action:** Create single source of truth in `sites/[tld]/[domain]/` structure

### 2. Domains Missing from API Routing

The 98 domains in site config but not in API routing need review:
- Should they route through the API worker?
- Or stay on their current platform (Vercel/Cloudflare Pages)?
- Or be deprecated?

### 3. Domains in API Routing but Not Site

The 61 domains in API routing but not documented in site config:
- Add to site config for documentation
- Or remove if no longer needed

### 4. Next Steps

1. **Fetch live domains from Cloudflare/Vercel APIs**
   - Set `CLOUDFLARE_API_TOKEN` and `CLOUDFLARE_ACCOUNT_ID`
   - Set `VERCEL_TOKEN`
   - Run full inventory with health checks

2. **Check domain health**
   - Nameserver configuration
   - HTTP response codes
   - SSL certificates

3. **Create `sites/` structure**
   ```
   sites/
   ├── do/
   │   ├── agents/
   │   ├── workflows/
   │   └── ...
   ├── mw/
   │   ├── db/
   │   └── api/
   └── services/
       └── api/
   ```

4. **Migrate to unified routing**
   - Start with healthy domains
   - Test routing logic
   - Document migration process
